# Research Assistant V2 信息架构

## 设计目标

页面围绕用户正在完成的科研工作组织，而不是围绕数据库表组织。每个页面只承担一个主要任务，并始终使用同一条业务主线：

`科研项目 → 实验计划 → 实验执行 → 过程记录 → 文件与数据`

## 领域边界

- 科研项目回答“这个课题要解决什么问题”，是实验计划和项目任务的最高层级。
- 实验计划回答“准备为什么做、按什么方案做”，保存目标、步骤、计划参数和样本要求。
- 实验执行回答“这一次真实怎么做”，保存实际日期、人员、分组、参数、样本、状态、总结和整体结论。
- 计划步骤只定义预期操作；创建实验执行时生成独立执行步骤快照，完成状态只属于该次执行。
- 过程记录回答“某个时间点做了什么、观察到什么、结果如何”，必须属于一次实验执行。
- 文件与数据属于产生它的过程记录；实验计划页只聚合查看和导出，不直接拥有散落的附件。

内部模型 `ExperimentBatch` 对应界面中的“实验执行”，`ExperimentRecord` 对应“过程记录”。数据库兼容名称不再出现在主要页面文案中。

## 全局导航

### 工作

- 总览：今天需要继续的任务、实验计划和最近过程记录。
- 任务：个人待办和项目任务。
- 科研项目：按研究问题组织实验计划、执行和汇报。
- 实验计划：查找实验计划，并从空白或步骤模板发起新计划。

### 资源

- 模板中心：独立创建、预览、编辑和调用步骤模板与记录模板。
- 样本：管理样本库存、计划用途和实际使用关系。
- 论文：管理投稿、返修和审稿事项。

### 输出

- 统计：查看任务、实验计划和过程记录结果概览。
- 汇报：选择实验证据生成周报 PowerPoint。

### 个人菜单

- 点击侧栏头像打开 API 预设、账号安全、管理员入口和退出登录。
- API 页面只维护多预设连接，不再显示单连接兼容表单或独立视觉模型等字段。
- 外观设置由页面右上角独立按钮打开。
- AI 只保留右下角悬浮助手，不在侧栏或独立导航中重复出现。

## 页面职责

### 科研项目

- 项目列表可直接创建项目、查看项目目标和计划/执行/记录数量。
- 项目详情聚合实验计划和每个计划下的实际执行。
- AI 可以创建或修改科研项目，但必须先显示字段差异并由用户确认。

### 实验列表

- 主任务是查找并打开实验计划。
- 支持空白创建和从步骤模板创建。
- 创建计划不会隐式生成实验执行或过程记录。

### 实验计划详情

- 概览：目标、项目、负责人、计划日期和状态。
- 实验方案：步骤、计划参数和样本要求。
- 实验执行：创建并打开每一次真实执行；跨执行过程记录只作为只读索引。
- 实验级导出位于页面顶部，按实验执行组织各条过程记录和附件清单。

### 实验执行详情

- 顶部展示状态、人员、实际日期和过程记录数量。
- 执行信息保存编号、重复类型、分组、实际日期、总结和整体结论。
- 执行步骤、实际参数、实际样本和过程记录都只属于当前执行。
- 新建过程记录时可以调用记录模板，并直接上传文件或整个文件夹。
- AI 在此页使用 `manage_batch` 边界管理执行信息、实际参数和过程记录。

### 过程记录详情

- 阅读：先展示日期、人员、条件、操作与观察、结果、结论和参数。
- 文件与数据：上传、分类、批量整理、外部路径索引和完整性检查。
- 编辑：修改草稿；定稿后必须填写修订原因并保留前后差异。
- 模板与修订：保存记录模板、查看已有模板和修订历史。

### API 预设

- 每个预设只填写名称、API URL、API Key、当前模型和提醒开关。
- “拉取模型”读取常见 OpenAI-compatible `/models` 响应。
- 模型列表使用视觉、推理、联网搜索和工具图标展示能力。
- 图标明确区分接口声明、名称推测、不支持和未知；兼容服务不会被误报为支持联网。

## 核心流程

### 记录一次真实实验

1. 创建或打开科研项目。
2. 创建实验计划，填写目标和方案。
3. 在实验计划页点击“新建实验执行”，系统会复制当前计划步骤作为本次执行快照；需要立即开做时再把状态改为“进行中”并填写实际开始日期。
4. 在执行页更新本次步骤，并填写实际参数和实际样本。
5. 逐条添加过程记录，并把文件与数据归入产生它的记录。
6. 完成后填写执行总结和整体结论，再从实验计划页统一导出。

### 使用 AI 写入页面

1. 打开右下角 AI 悬浮助手。
2. 选择实验计划及其中的具体实验执行、知识库和需要发送的附件上下文。
3. 输入创建或修改项目、规划实验、新建执行或整理当前执行等指令。
4. 检查 AI 返回的来源和修改前后差异。
5. 人工确认后写入；页面内容已变化时旧提案自动失效。

## 交互约束

- 不使用卡片套卡片表达层级。
- 页面最多一个主要按钮，次要操作进入工具栏或折叠区。
- 折叠只用于低频补充信息，不承担页面主导航。
- 批量工具在用户选中内容后才突出显示。
- 桌面端内容宽度和工具栏保持稳定；移动端操作换行，不遮挡标题和正文。
- 对模型能力、AI 推断、剂量、临床解释和统计结论使用保守文案并保留人工核验提示。
