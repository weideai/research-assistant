# 更新日志

## 2026-07-24 · V2 科研工作流与 AI 配置收口

### 本次目标

本轮把产品主线统一为：

`科研项目 → 实验计划 → 实验执行 → 过程记录 → 文件与导出`

实验计划回答“准备做什么”，实验执行表示“一次实际开展”，过程记录只描述该次执行中“发生了什么”。内部旧模型名暂时保留以兼容已有数据库，但用户界面、AI 提示和导出文件统一使用“实验执行 / 过程记录”。

### API 预设与模型能力

- API 设置只保留可命名、保存和切换的预设，不再展示容易混淆的单连接兼容表单。
- 移除视觉、嵌入和图像模型的手工输入框；一个预设选择一个主模型，能力由模型列表和接口元数据自动描述。
- 支持从兼容服务的 `/models` 接口拉取模型，兼容 `data`、`models`、嵌套对象、直接数组、字符串模型项以及 `id`、`name`、`model` 等常见字段。
- 模型列表以小图标展示视觉、推理、工具调用和联网搜索能力，并区分“接口声明、保守推测、明确不支持、未知”，不会把兼容服务的模型名称误判成已支持联网。
- 支持 camelCase 能力元数据，例如 `supportsFunctionCalling` 和 `inputModalities`。
- 账号中已启用的默认预设是 AI 唯一连接来源；没有可用预设时保持未连接，不再回落到环境变量。
- 模型能力快照随预设持久化，并同时驱动 API 页面图标和助手联网开关；停用的默认预设不再显示为“当前使用”。
- 旧 `ApiSetting` 会迁移或合并到预设后清理；已有预设不会被旧配置替换。API Key 继续加密保存，页面只显示掩码。

### 导航与个人菜单

- 主导航突出科研项目、实验计划、模板、样本、论文、统计和汇报，不再放置独立 AI 页面。
- AI 助手只保留全局悬浮框，可在当前科研页面继续对话和发起经确认的操作。
- API 设置、账号安全和管理员入口集中到侧栏底部个人头像菜单；外观切换保持独立入口，避免与助手按钮混在一起。
- 模板中心保持一级入口，步骤模板和记录模板都能在调用前直接创建、预览、编辑、复制和删除。

### AI 科研助手

- AI 现在可以创建和修改科研项目，也可以从当前实验计划新建一次实验执行；所有写入先展示字段差异，用户确认后才写入数据库，并保留撤销信息。
- AI 可在科研项目、实验计划、实验执行和过程记录上下文中管理字段、步骤、参数、样本使用、过程记录和附件元数据。
- 新建实验计划时会在差异确认区明确选择所属科研项目；账号下有多个项目时不再静默写入最早创建的项目。
- 跨页面继续旧会话时，发送前预览与实际生成共用当前页面的已验证上下文，不会出现“预览 A、实际修改 B”。
- 项目页会默认选中该项目下的活动实验历史，可直接比较执行、总结过程记录或生成项目周报。
- AI 历史范围可先按实验计划全选，也可精确勾选某几次实验执行；提示上下文和引用会携带执行编号、实际参数及执行 ID，不同重复执行不再混成一组记录。
- 实验执行修改增加枚举、布尔值、日期区间、跨执行记录、软删除父级和撤销冲突校验。
- AI 快照覆盖过程记录生命周期、定稿时间、来源消息、参数、附件状态和修订关系；页面在保存后又发生变化时会停止撤销，避免覆盖新内容。
- 新建科研项目或实验计划后会跳转到对应页面；过程记录和文件操作会定位到实际发生变化的工作区。

### 实验执行与过程记录

- 实验计划详情只承担“概览 / 实验方案 / 实验执行”，不再把计划、执行和记录平铺在同一页面。
- 新建实验计划不会隐式产生空执行；用户开始实际工作时再创建一次实验执行。
- 每次实验执行独立拥有执行编号、重复类型、实际日期、实际参数、样本使用、状态、摘要和结论。
- 实验计划中的步骤只定义方案，不再保存共享完成状态；新建实验执行时会复制一份独立执行步骤快照，本次完成状态、执行人和完成日期只影响当前执行。
- 新增过程记录、记录模板调用、过程时间线和批量管理集中到所属实验执行页面。
- 记录模板在模板中心和详情页直接选择目标实验执行并打开新增过程记录表单，不再经过实验计划页二次选择。
- 默认执行编号统一为 `RUN-01`；创建过程记录时保留校验失败、普通保存和带文件保存各自正确的返回位置。
- 每条活动过程记录必须归属于当前实验计划的一次活动执行；跨计划、跨执行和已删除父级请求会被拒绝。
- 记录详情明确返回所属实验执行；文件继续归入产生它的具体过程记录。

### 导出一致性

- JSON 导出升级为 schema v3；计划步骤只保留方案字段，每次实验执行独立导出步骤快照和完成状态，并继续保留带执行编号的平铺 `records` 兼容视图。
- 无法归档的活动旧记录会进入明确的 `unassigned_records / HISTORY-UNASSIGNED` 兼容分组，常规导出不会静默丢数据。
- Markdown、Word 和 Excel 均按“实验执行 → 过程记录 → 文件”组织；Word 标题和 Excel 工作表统一使用新术语。
- Excel 新增“实验执行”工作表，“过程记录”“过程记录参数”和“附件清单”均包含执行编号。
- ZIP 中的 Markdown、JSON、文件清单和附件目录共用同一份过滤后快照，附件路径按执行编号分层。
- `.ralab` 项目包升级为 schema v2，保存每次执行的步骤快照；导入仍兼容旧 schema v1 项目包。
- 所有格式过滤已移入回收站的过程记录和附件；ZIP 不再打包软删除文件。
