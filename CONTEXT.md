# 医学科研工作台

面向医学研究生的个人科研工作领域，重点是让单个科研人员持续组织、记录、检索和复用自己的实验工作。

## Language

**个人科研工作台**:
由单个科研人员独立拥有和使用的科研信息空间，是本产品未来六个月的首要产品定位。
_Avoid_: 实验室管理系统、多人协作平台、临床数据平台

**科研项目**:
围绕同一研究问题组织目标、实验计划、进度和汇报的最高业务范围。
_Avoid_: 研究项目、论文、文件夹

**实验计划**:
描述一个实验为什么做、如何做以及预期使用哪些条件的可复用方案；同一计划可以产生多次实验执行。
_Avoid_: 实验执行、过程记录

**实验执行**:
按照某个实验计划开展的一次实际工作，独立拥有执行编号、实际日期、实际参数、样本使用、状态、总结和结论；同一实验执行可以包含多条过程记录。代码中的 `ExperimentBatch` 是这一概念的兼容名称。
_Avoid_: 实验计划、执行批次、实验记录

**计划步骤**:
实验计划中按顺序定义的操作方案，可以指定计划执行人和计划日期，但不表达任何一次真实执行是否完成。
_Avoid_: 已完成步骤、执行步骤、过程记录

**执行步骤**:
某次实验执行创建时从计划步骤生成的独立快照，记录该次执行自己的完成状态、实际执行人和完成日期。
_Avoid_: 计划步骤、共享完成状态、过程记录

**过程记录**:
实验执行中一条有日期的操作、观察或结果日志，文件归入产生它的记录；一条过程记录必须且只能属于一次实验执行。代码中的 `ExperimentRecord` 是这一概念的兼容名称。
_Avoid_: 实验执行、执行总结、脱离执行的实验记录

**计划样本用途**:
实验计划中预计使用的样本、角色和用量，不代表样本已经被实际消耗。
_Avoid_: 实际样本使用、库存流水

**实际样本使用**:
某次实验执行真实使用的样本、角色和用量，是样本追溯和库存判断的依据。
_Avoid_: 计划样本用途、样本要求

**步骤模板**:
可独立创建、预览、编辑和调用的实验步骤方案；调用时可以追加到实验计划或替换其现有步骤。
_Avoid_: 实验模板、记录模板、已有实验的隐藏副本

**记录模板**:
可独立创建、预览、编辑和调用的记录正文及结构化参数方案；调用时只预填新记录，不直接生成已完成记录。
_Avoid_: 实验模板、步骤模板、固定表单

**草稿记录**:
实验过程中尚未确认完成、可以自由编辑的过程记录。
_Avoid_: 定稿记录、原始记录

**定稿记录**:
由用户确认完成并锁定原始内容的过程记录；后续纠错只能通过修订表达。
_Avoid_: 草稿、只读记录

**修订**:
在保留定稿记录原文的前提下，对其进行的有时间、原因和内容差异的更正或补充。
_Avoid_: 覆盖修改、重新保存

**托管文件**:
由工作台复制并管理其物理内容的实验附件。
_Avoid_: 外部数据链接

**外部数据链接**:
指向用户自行保管的大型原始数据的索引，仅保存路径、清单和完整性信息，不复制或删除原文件。
_Avoid_: 托管文件、云同步

**知识资料**:
论文、SOP、说明书、仪器手册或方法笔记等参考来源，与用户实际产生的实验历史严格分开。
_Avoid_: 实验历史、实验结果

**科研 Skill**:
经过审核的提示词、输入约束、输出结构和版式资源组合；用户自定义 Skill 不允许执行脚本。
_Avoid_: 任意插件、可执行脚本包
